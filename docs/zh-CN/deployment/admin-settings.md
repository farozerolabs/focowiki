# Admin 配置

在 Admin UI 知识库页面打开**设置**。保存后的值会应用到后续请求和后台任务，并在服务重启后继续生效。

## API 限流

| 字段 | 含义 | 推荐值 |
| --- | --- | --- |
| 管理员登录 / 最大请求数 | 一个窗口内允许的登录尝试次数。 | `8`；公网部署可使用 5 到 10。 |
| 管理员登录 / 窗口秒数 | 登录计数窗口长度。 | `900`。 |
| Admin API / 最大请求数 | 一个窗口内允许的 Admin UI API 请求次数。 | `600`。 |
| Admin API / 窗口秒数 | Admin API 计数窗口长度。 | `60`。 |
| Developer OpenAPI / 最大请求数 | 一个窗口内允许的 Developer OpenAPI 请求次数。 | `1200`，再按服务器容量调整。 |
| Developer OpenAPI / 窗口秒数 | Developer OpenAPI 计数窗口长度。 | `60`。 |

上传登记遵循需要鉴权的上传会话契约，不设置独立的产品级上传频率和逻辑文件数量配额。反向代理和存储服务仍可能执行 Focowiki 之外的基础设施限制。

## Worker

Worker 配置控制来源处理、持久化派发、重试、记录保留和异步删除。硬上限与恢复值组成迟滞区间：压力达到硬上限后暂停派发，降到较低的恢复值后继续派发。派发暂停期间上传登记继续运行。

| 字段 | 含义 | 推荐值 |
| --- | --- | --- |
| 来源文件并发 | 同时处理的来源文件数量。 | 完成数据库和存储延迟评估后，8C/32G 服务器使用 8 到 16。 |
| 来源对象读取并发 | 单个来源 Worker 同时读取的来源 Markdown 对象数量。 | 8 到 16，且不高于来源文件并发。 |
| 图查询并发 | 单个来源 Worker 同时执行的图候选数据库查询数量。 | 8 到 16，且不高于来源文件并发。 |
| 数据库变更并发 | 单个来源 Worker 同时执行的来源处理数据库变更组数量。 | 4 到 8，且不高于来源文件并发。 |
| 领取批次大小 | 每轮领取的来源任务数量。 | 不低于来源文件并发；8C/32G 服务器使用 32。 |
| 生成批次大小 | 单个有界 generation 输入批次提交的来源记录数。 | 50 到 200。 |
| 轮询间隔毫秒 | 两次队列检查之间的等待时间。 | 1000 到 3000 毫秒。 |
| 锁 TTL 秒数 | 已领取任务锁的有效时间。 | 长于正常处理时间，常用 900 秒。 |
| 心跳间隔毫秒 | 运行中任务刷新领取状态的间隔。 | 10000 到 30000 毫秒。 |
| 任务最大尝试次数 | 任务进入死信前允许的尝试次数。 | `3`。 |
| 任务重试延迟毫秒 | 临时失败后再次执行前的等待时间。 | 30000 到 120000 毫秒。 |
| 来源队列硬上限 | 达到该排队数量后暂停新任务派发。 | 5000 到 20000。 |
| 来源队列恢复值 | 低于该排队数量后允许恢复派发。 | 硬上限的 50% 到 70%。 |
| 来源队列硬等待秒数 | 最早排队任务达到该等待时间后暂停派发。 | 3600 到 7200 秒。 |
| 来源队列恢复等待秒数 | 最早排队任务低于该等待时间后允许恢复派发。 | 硬上限约一半。 |
| 关闭等待毫秒 | Worker 有序关闭时允许的收尾时间。 | 30000 到 120000 毫秒。 |
| 完成任务保留天数 | 完成任务记录的保留时间。 | 7 到 30 天。 |
| 失败任务保留天数 | 失败任务记录的保留时间。 | 30 天或更长。 |
| 死信任务保留天数 | 死信任务记录的保留时间。 | 90 天。 |
| 保留清理批次大小 | 单个维护分页删除的任务记录行数。 | 500 到 2000。 |
| 清理并发 | 同时处理的异步删除任务数。 | 多数部署使用 `1`。 |
| 清理数据库批次大小 | 单个清理分页删除的数据库行数。 | 500 到 2000。 |
| 清理对象批次大小 | 单次请求删除的 S3 对象数。 | `1000`，最大值为 1000。 |
| 清理最大尝试次数 | 异步删除允许的尝试次数。 | `3`。 |
| 清理重试延迟毫秒 | 异步删除临时失败后的等待时间。 | 60000 到 300000 毫秒。 |
| 清理失败保留天数 | 清理失败证据的保留时间。 | 30 天。 |
| 版本对象清理 | 开启 S3 版本对象删除。 | 仅在存储生命周期要求时开启。 |

## 发布

发布过程会写入不可变生成对象、更新受影响的投影分片、校验变更闭包，并原子切换当前生效的 generation。当前 generation 冻结后，后续上传可以继续进入一个合并的后继 generation。

| 字段 | 含义 | 推荐值 |
| --- | --- | --- |
| 模式 | `batch`、`per_file` 或 `manual` 发布调度。 | 大批量导入使用 `batch`，快速可见使用 `per_file`。 |
| 批次大小 | 达到该来源变更数量后，批量 generation 立即具备执行条件。 | 100 到 500。 |
| 间隔秒数 | 从 generation 创建时间计算的最长批量等待时间。 | 120 到 600 秒。 |
| 角色并发 | 发布角色同时处理的任务数。 | 完成数据库和 S3 容量评估前使用 `1`。 |
| Generation 组装并发 | 同时组装的有界 generation 输入页数量。 | 1 到 2，且不高于角色并发。 |
| 投影分区并发 | 同时处理的独立物理投影分区数量。 | 8 到 16，且不高于影响项并发。 |
| 生成对象写入并发 | 同时上传并校验的不可变生成对象数量。 | 8 到 16，且不高于投影分区并发。 |
| 目录生成并发 | 同时生成的独立目录导航输出数量。 | 4 到 8，且不高于投影分区并发。 |
| 领取批次大小 | 每轮领取的发布任务数。 | 1 到 4，且不低于角色并发。 |
| 影响项批次大小 | 单个有界分页处理的投影影响项数。 | 100 到 500。 |
| 待投影文件硬上限 | 达到该待投影文件数后暂停来源派发。 | 2000 到 10000。 |
| 待投影文件恢复值 | 低于该待投影文件数后恢复来源派发。 | 低于硬上限。 |
| 待投影硬等待秒数 | 最早待投影文件达到该时间后暂停来源派发。 | 900 到 3600 秒。 |
| 待投影恢复等待秒数 | 最早待投影文件低于该时间后恢复来源派发。 | 低于硬上限。 |
| 待处理影响项硬上限 | 达到该投影影响项数量后暂停来源派发。 | 20000 到 100000。 |
| 待处理影响项恢复值 | 低于该投影影响项数量后恢复来源派发。 | 低于硬上限。 |
| Generation 保留天数 | 非活动 generation 引用进入垃圾回收前的保留时间。 | 7 到 30 天。 |
| 索引分片大小 | 单个稳定搜索机器分片包含的记录数。 | 1000 到 5000。 |
| 链接索引分片大小 | 单个稳定链接机器分片包含的记录数。 | 1000 到 5000。 |
| Manifest 分片大小 | 单个稳定 Manifest 机器分片包含的记录数。 | 1000 到 5000。 |
| 图边分片大小 | 单个稳定图机器分片包含的边数。 | 5000 到 20000。 |
| 图候选上限 | 单个有界投影操作评估的关系候选数。 | 100 到 300。 |
| 图维护批次大小 | 单个维护分页刷新的图记录数。 | 200 到 1000。 |
| 根摘要上限 | 根目录有界摘要包含的条目数。 | 200 到 1000。 |
| 目录索引单页条目数 | 单个目录导航页包含的直接条目数。 | 100 到 500。 |
| 目录索引单页字节数 | 单个目录导航页的 UTF-8 字节边界。 | 65536 到 262144。 |
| 日志最大条数 | `log.md` 保留的最近 generation 变更数。 | 50 到 200。 |
| 日志最大字节数 | `log.md` 的 UTF-8 字节边界。 | 65536 或更高。 |

## 图关系

图关系配置控制基于正文的文件关系发现、图搜索、探索边界、生成图分片和短期查询缓存。

| 字段 | 含义 | 推荐值 |
| --- | --- | --- |
| 图候选上限 | 关系生成时考虑的候选文件数。 | 100 到 300。 |
| 接受关系上限 | 每个文件保留的已接受关系数。 | 20 到 80。 |
| 默认搜索深度 | OpenAPI 未传 `depth` 时使用的图扩展深度。 | `1`。 |
| 最大搜索深度 | OpenAPI 接受的最大图扩展深度。 | `2`。 |
| 默认搜索扇出 | 每一跳默认继续读取的相关文件数。 | `10`。 |
| 最大搜索扇出 | 每一跳允许继续读取的相关文件上限。 | `25`。 |
| 模型关系复核 | 允许生效模型复核关系候选。 | 模型服务稳定时开启。 |
| 图发布分片大小 | 单个生成分片包含的图节点和图边数。 | 5000 到 20000。 |
| 图缓存 TTL 秒数 | 图搜索和图扩展的 Redis 缓存时间。 | 5 到 60 秒。 |
| 通用短语阈值 | 通用短语过滤使用的最小归一化长度。 | `4`。 |

## 维护

维护配置用于有界对账 Focowiki 管理的生成对象。对账只在维护 Worker 中运行，不扫描来源上传、上传会话对象、其他 prefix 或用户管理的存储路径。

| 字段 | 含义 | 推荐值 |
| --- | --- | --- |
| 存储对账 | 开启有界生成对象对账。 | 正常部署保持开启。 |
| 扫描间隔秒数 | 两次完整对账周期之间的时间。 | `21600` 秒。 |
| 扫描批次大小 | 单个有界分页读取的生成对象元数据数量。 | `500`，最大 `1000`。 |
| 删除批次大小 | 单个有界批次删除的已确认孤立对象数量。 | `100`，最大 `1000`。 |
| 隔离宽限秒数 | 未登记候选在具备删除条件前至少保留的时间。 | `86400` 秒或更长。 |
| 确认轮次 | 具备删除条件前需要完成的发现轮次。 | `2` 或更高。 |
| 最大尝试次数 | 单个候选保留的删除尝试次数。 | `5`。 |
| 重试延迟毫秒 | 对账临时失败后的等待时间。 | `30000` 到 `300000` 毫秒。 |
| 迁移回填并发 | 兼容优化迁移期间同时处理的有界来源与投影页数量。 | 1 到 2。 |
| 投影压缩并发 | 同时压缩的独立投影分区数量。 | 1 到 2。 |

状态区域只展示扫描、隔离、删除、重试和已登记但存储缺失的汇总数量，不返回对象键、校验值、存储凭证、SQL、Redis key 或内部 Worker 载荷。

## 模型

模型辅助为选填能力。没有生效模型时，来源处理会继续生成确定性元数据、导航、搜索和图输入。同一时间只能有一个生效模型。

| 字段 | 含义 | 推荐值 |
| --- | --- | --- |
| 显示名称 | Admin UI 中显示的名称。 | 写清服务商和用途。 |
| API 模式 | `responses` 或 `chat_completions` 服务商协议。 | 与服务商端点保持一致。 |
| Base URL | OpenAI 兼容 API 地址。 | 服务商要求时包含 `/v1`。 |
| API key | 只由后端发送的访问凭证。 | 使用独立权限并定期轮换的 key。 |
| 模型名称 | 服务商模型标识。 | 与服务商文档完全一致。 |
| 上下文窗口 tokens | 模型上下文容量。 | 使用服务商公布的真实上限。 |
| 请求最大超时毫秒 | 单次模型请求的总时间上限。 | 长文档使用 600000 毫秒或更高。 |
| 请求空闲超时毫秒 | 模型响应无活动的最长时间。 | 120000 到 300000 毫秒。 |
| 建议生成并发 | 同时执行的模型建议请求数。 | 从 1 到 2 开始。 |
| 临时错误重试延迟毫秒 | 服务商临时失败后的等待时间。 | 60000 毫秒。 |
| 请求最小间隔毫秒 | 两次模型请求之间的最小间隔。 | 稳定服务使用 0，严格限流使用 1000 到 5000 毫秒。 |

模型创建后不再显示完整 API key。暂停模型后，新任务不会再选择该模型。运行中的任务仍引用该模型时，删除操作会被阻止。

## 生效与观察

运行时配置会持久保存。需要稳定快照的运行中任务会继续使用领取任务时的配置，后续领取和请求使用新值。端口、域名、基础设施凭证、数据库连接池、存储凭证和日志目录属于启动配置，继续保存在 `.env` 中，修改后需要重启对应服务。
