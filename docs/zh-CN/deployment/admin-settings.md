# Admin 配置

打开 Admin UI，在知识库首页点击设置图标，即可修改运行时配置。保存后，新的配置会用于后续请求和后续后台任务，不需要编辑 `.env` 或重启服务。

首次部署时，系统会使用默认值初始化这些配置。管理员保存配置后，后续运行会优先使用页面中保存的值。

模型 API key 保存后会受到保护。页面只展示密钥标识，不会再次展示完整密钥。部署数据保留完整时，服务重启后已保存 key 仍可使用。维护时如果删除了保护数据，需要在本页重新录入受影响的模型 API key。

## 限流

限流用于保护管理员登录、Admin API、上传请求和 Developer OpenAPI。每个分组都有相同的两个字段。

| 分组 | 字段 | 含义 | 推荐值 |
| --- | --- | --- | --- |
| 管理员登录 | 最大请求数 | 一个计数窗口内允许的登录尝试次数上限。 | `8`，公网部署可使用 5 到 10。 |
| 管理员登录 | 窗口秒数 | 登录尝试次数的计数窗口长度。 | `900`。 |
| Admin API | 最大请求数 | 一个计数窗口内允许的 Admin UI API 请求次数上限。 | `600`。 |
| Admin API | 窗口秒数 | Admin UI API 请求次数的计数窗口长度。 | `60`。 |
| 上传 | 最大请求数 | 一个计数窗口内允许的 Markdown 上传请求次数上限。 | `20`。 |
| 上传 | 窗口秒数 | Markdown 上传请求次数的计数窗口长度。 | `3600`。 |
| Developer OpenAPI | 最大请求数 | 一个计数窗口内允许的 Developer OpenAPI 请求次数上限。 | `1200`，再按服务器容量和真实流量调整。 |
| Developer OpenAPI | 窗口秒数 | Developer OpenAPI 请求次数的计数窗口长度。 | `60`。 |

运行时限流应结合反向代理、Cloudflare 或其他边缘层限流一起调整。

## 后台处理

后台处理配置控制来源文件的处理并发、任务领取、重试、排队保护和任务记录保留。正在运行的任务会继续完成当前步骤。新的任务会使用最新保存值。

| 字段 | 含义 | 推荐值 |
| --- | --- | --- |
| 来源文件并发 | 同时处理的来源文件数量。 | 8C/32G 服务器推荐 2 到 4。 |
| 领取批次大小 | 每轮领取的任务数量。 | 10 到 50，并接近实际并发。 |
| 轮询间隔毫秒 | 后台服务检查待处理任务的间隔。 | 1000 到 3000 毫秒。 |
| 锁 TTL 秒数 | 任务锁的有效时间。 | 长于单文件正常处理时间，常用 900 秒。 |
| 心跳间隔毫秒 | 运行中任务刷新心跳的间隔。 | 10000 到 30000 毫秒。 |
| 任务最大尝试次数 | 任务进入死信前的最大尝试次数。 | 3 次。 |
| 任务重试延迟毫秒 | 失败任务再次重试前的等待时间。 | 30000 到 120000 毫秒。 |
| 全局队列上限 | 全局排队任务上限。 | 大服务器推荐 5000 到 20000。 |
| 知识库队列上限 | 单个知识库的排队任务上限。 | 低于全局上限。 |
| 队列最大等待秒数 | 最早排队任务超过该时长后放慢上传。 | 3600 到 7200 秒。 |
| 重试等待秒数 | 队列繁忙时建议等待的时间。 | 30 到 300 秒。 |
| 关闭等待毫秒 | 后台服务关闭时允许任务收尾的时间。 | 30000 到 120000 毫秒。 |
| 完成任务保留天数 | 完成任务记录保留天数。 | 7 到 30 天。 |
| 失败任务保留天数 | 失败任务记录保留天数。 | 30 天或更长。 |
| 死信任务保留天数 | 死信任务记录保留天数。 | 90 天。 |
| 保留清理批次大小 | 每次清理任务记录的行数。 | 500 到 2000。 |
| 清理并发 | 同时处理的后台清理任务数量。 | 多数部署使用 1。 |
| 清理数据库批次大小 | 单次后台清理批次处理的数据库行数。 | 500 到 2000。 |
| 清理对象批次大小 | 单次后台清理请求处理的存储对象数量。 | 1000，且不能高于 1000。 |
| 清理最大尝试次数 | 后台清理工作的最大尝试次数。 | 3。 |
| 清理重试延迟毫秒 | 后台清理遇到临时失败后的重试等待时间。 | 60000 到 300000 毫秒。 |
| 清理失败保留天数 | 后台清理失败记录保留天数，用于维护排查。 | 30。 |
| 版本对象清理 | 后台清理时要求存储版本删除能力。 | 部署明确使用版本化存储清理时再开启。 |

队列上限需要支撑计划内导入量，同时保留管理后台和 OpenAPI 的响应能力。增加并发前先观察 CPU、内存、存储速度、文件处理速度和模型响应速度。

## 发布

发布会把处理完成的文件放入当前文件树，并让 Developer OpenAPI 可以读取。文件处理完成后，生成文件可能还需要等待发布后才可见。

| 字段 | 含义 | 推荐值 |
| --- | --- | --- |
| 模式 | 发布策略。`batch` 会批量发布完成文件，`manual` 等待明确的发布动作，`per_file` 在每个文件完成后发布。 | 大知识库推荐 `batch`，需要快速可见用 `per_file`，需要人工控制用 `manual`。 |
| 批次大小 | 单次发布任务包含的文件数量。 | 100 到 500。 |
| 间隔秒数 | 批量发布之间的最小间隔。 | 120 到 600 秒。 |
| 索引分片大小 | 搜索索引每个分片的条目数。 | 1000 到 5000。 |
| 链接索引分片大小 | 链接索引每个分片的条目数。 | 1000 到 5000。 |
| Manifest 分片大小 | Manifest 每个分片的条目数。 | 1000 到 5000。 |
| 图边分片大小 | 图关系边每个分片的数量。 | 5000 到 20000。 |
| 图候选上限 | 生成关系时参与候选的文件数量。 | 100 到 300。 |
| 图维护批次大小 | 每轮图关系维护刷新的文件数量。 | 200 到 1000。 |
| 根摘要上限 | 根目录摘要和索引展示的条目上限。 | 200 到 1000。 |
| 日志最大条数 | `log.md` 保留的最近更新条数。 | 50 到 200。 |
| 日志最大字节数 | 生成的 `log.md` 最大字节数。 | 活跃知识库推荐 65536 或更高。 |

文件处理列表会同时展示来源文件处理状态和生成结果可见状态。文件处理完成且生成结果可见时，该文件可被稳定读取。

## 上传和生成

上传和生成配置控制 Markdown 上传大小、请求规模和生成工作批次。

| 字段 | 含义 | 推荐值 |
| --- | --- | --- |
| 最大上传字节数 | 单次上传请求允许的总字节数上限。 | `10485760`，即 10 MB；小型部署可降低。 |
| 最大上传文件数 | 单次上传请求允许的 Markdown 文件数量上限。 | 8C/32G 服务器推荐 50。 |
| 生成批次大小 | 生成、图关系、索引和发布工作每批处理的数据量。 | 8C/32G 服务器推荐 100。 |
| 文件处理并发 | 单个后台任务中的文件处理并发数。 | 大规模导入推荐 1，更稳定。 |
| 存储并发 | 上传源文件同时写入 S3 兼容存储的数量。 | 推荐 4，S3 稳定时可使用 6。 |

上传只接受 Markdown 文件。来源文件可以先上传成功，处理和发布会在后台继续执行。

## 模型

模型辅助是可选能力。没有生效模型时，上传会继续执行确定性 OKF 生成，并跳过模型建议。

| 字段 | 含义 | 推荐值 |
| --- | --- | --- |
| 显示名称 | 管理后台展示的模型名称。 | 写清提供商和用途。 |
| API 模式 | 模型请求使用的接口协议。`responses` 使用 Responses API structured outputs，`chat_completions` 使用 Chat Completions JSON object output。 | 提供 `/responses` 的服务商使用 `responses`；提供 `/chat/completions` JSON 输出的服务商使用 `chat_completions`。 |
| Base URL | OpenAI 兼容 API 地址。 | 服务商要求时包含 `/v1`。 |
| API key | 模型服务 API key。 | 使用独立权限 key，并定期轮换。 |
| 模型名称 | 发送给模型服务的模型标识。 | 与服务商文档完全一致。 |
| 上下文窗口 tokens | 模型上下文窗口长度。 | 填写模型真实上下文上限。 |
| 请求最大超时毫秒 | 单次请求最大等待时间。 | 长文档推荐 600000 毫秒或更高。 |
| 请求空闲超时毫秒 | 等待模型输出时的空闲超时。 | 120000 到 300000 毫秒。 |
| 建议生成并发 | 模型建议生成的并行请求数。 | 先用 1 到 2，稳定后再调高。 |
| 临时错误重试延迟毫秒 | 临时失败后的重试等待时间。 | 60000 毫秒。 |
| 请求最小间隔毫秒 | 模型请求之间的最小间隔。 | 稳定服务商可用 0，限流严格时推荐 1000 到 5000 毫秒。 |

设为生效的模型会用于新的来源文件任务。暂停模型会保留配置，并阻止新的任务使用该模型。模型仍被运行中的任务使用时，删除操作会被阻止。

## 运维说明

- 配置保存后会持续生效，服务重启后仍会使用已保存值。
- 删除部署数据会同时清除已保存配置。
- 配置只在 Admin UI 中管理，不通过 Developer OpenAPI 暴露。
- 运行时变更作用于后续请求和后续后台任务。部分值会在下一次任务或下一轮检查后影响长时间运行的工作。
