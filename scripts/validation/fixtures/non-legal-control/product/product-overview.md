# Atlas Workspace Overview

Atlas organizes projects into workspaces. Each workspace contains members, data connections, saved views, and automation rules.

## Roles

Owners manage billing and access. Editors create and update project content. Viewers can inspect published material without changing it.

## First-run workflow

1. Create a workspace.
2. Invite at least one editor.
3. Connect a data source.
4. Publish the first saved view.

For production rollout steps, read [Change Management](../operations/change-management.md).
